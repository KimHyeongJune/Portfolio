2022/09/06 ~ 2022/09/07
Tmax Super Week 2022 
1. SuperApp 소개
 - App의 통합뿐만 아니라 플랫폼의 통합.
 - 개발자의 도움 없이 자신(개인, 소상공인, 기업, 국가 등)의 아이디어로 앱을 만들어 낼 수 있도록 하는 것이 목표
2. 혼돈의 클라우드 재정의 (TCP : Tmax Cloud Platform)
 - 하이브리드 클라우드 문제 : 클라우드 사용량 증가로 스토리지가 늘어나며 점점 더 많은 비용을 지불하는 현상을 막기 위하여 자사 데이터 센터와 클라우드 데이터를 동시에 이용하고 있음.
 - 오픈소스, 쿠버네티스를 사용한 유연성을 추구하는 쪽으로 발전하고 있음
  -> 오픈소스를 사용하면 에러에 대한 책임이 따르고 오픈소스를 수정할 수 있는 인력 부족
 - MSA (Micro Service Architecture) : 거대화 된 서비스를 잘게 쪼개어 운영하는 방식. 
  ->신규서비스 추가가 필요할 때 쪼개진 서비스의 데이터를 이용하는 경우 데이터 정합성에 문제가 발생.
 - SaaS (Software as a Service) : 만들어진 애플리케이션을 가져다가 쓰는 것. 
  -> 커스텀한 개발이 불가능하고, 다른 SaaS 서비스로의 연결이 불가능.
 - 데이터 가상화 (DMP, Data Management Platform)를 통하여 다양한 데이터베이스를 마치 하나의 데이터베이스인것처럼 사용할 수 있도록 함.
 - 티맥스의 SuperUX, SuperDX, SuperOS, SuperCloud를 통한 MaaS(Management as a Service)를 제공.
3. SuperApp 세상의중심, SuperTibero (Database)
 - Digital Transformation : 클라우드 기반의 데이터, 문화의 전환
 - 현재 DBMS의 클라우드 전환은 중요 업무보다는 중요성이 낮은 업무 전환이 대부분
 - Master-Slave : Master에서 트랜잭션이 일어날 경우 Slave로 복제되어 조회성 업무는 빠르게 처리되나 대량 트랜잭션이 일어날 경우 Master에 부하가 몰리게 됨.
 - 업무 중심으로 파편화 된 DB를 가상화 데이터베이스로 제공하여 통합 및 관리.
 - 현재 DBMS 이슈
  ㄱ. 확장과 성능 : 폭증하는 데이터와 그 데이터를 처리하는 속도 저하
  ㄴ. 연계와 통합 : 개발 방식과 모델이 달라 통합 처리 불가
  ㄷ. 개발과 활용 : 데이터 특성에 따라 환경이 달라 앱 개발 난이도 증가
 - 1티어 DBMS : DBMS가 단독으로 데이터 처리와 저장, 트랜잭션을 모두 관리.
   2티어 DBMS : DB가 데이터 처리를 담당하고 스토리지가 데이터 저장. 따라서 DB와 스토리지의 독립적인 확장이 가능하고 각 DB와 스토리지 간에도 독립적인 동작 구현 가능.
 - 데이터 타입별로 나누어진 데이터 형태를 통합 SQL 엔진을 통해 SQL 실행 시 동일한 데이터 형태로 나타내는 기능 제공
4. TmaxOffice, SuperOffice 
 - 코로나 팬더믹 이후 비대면 시대로 들어서며 문서의 사용량이 급격히 증가
 - 문서에 대한 보안, 관리, 환경, 비용에 대한 필요성이 대두됨.
 - 이러한 문제들은 파일 기반의 오피스 구조적 한계 대문에 발생.
 - 클라우드 웹 오피스가 개발되며 보안, 저장, 동시작업 등 많은 부분이 해결되었으나 오피스의 본직적인 기능인 다양한 기능과 속도 저하 등으로 사용자에게 채택되지 못하고 있음.
 - TmaxOffice의 장점
  ㄱ. 파일 기반 방식을 DBMS 저장 방식으로 변경
  ㄴ. 웹브라우저 기반을 SuperApp 기반으로 변경
  ㄷ. 문서생산도구를 문서 활용을 고려한 관리도구의 역할로 확장
 - 문서의 DB화 : 비정형 데이터를 반정형 데이터로 변환하여 객체 단위로 DB 저장.      이로인해 PC에 파일이 없으므로 랜섬웨어 감염이 불가능.
   또한 객체 단위로 저장되기 때문에 객체별 권한 부여나 검색이 가능함.

  ㄷ. 서비스
  ㄹ. 경영관리와 인사 : 내 회사에 맞는 앱을 SI 개발없이 쉽게 추가 기능 개발. 개발이 아닌 구현으로..

12. 관람 후기
9월6일 ~ 9월7일 
이틀동안 Tmax의 SuperWeek를 온라인으로 관람했다.
가서 봤으면 참 좋겠지만.. 나는 부산에 있고 일 때문에도 오프라인 참석하기도 어렵다.

Tmax 는 그래도 Jeus 서버 때문에 프로그래머를 시작한 이래로 줄곧 들어왔고, 지금 직장에 들어온지 얼마 안되었을 때 부장님과 함께 TmaxOS 소개 세미나를 참가하게 된 적이 있어 생소한 그룹은 아니었다.
그동안 Tmax 그룹은 당장의 수익만을 쫓는 기업이 아닌 미래를 준비하는 기업으로 바뀐 것 같았다.
물론 그 변화 과정에서 많은 개발자들도 빠져나가고, 대량 이직으로 인한 어려움도 겪은듯 하지만 적어도 현 상황에 안주하며 땜질식 프로그램을 만드는 기업보다는 나아보였다.
여러 세션에서 각 계열사마다 자신들의 노력과 결과물을 보여주기 위해 정말 준비를 많이 한 것 같았고, 발표하는 사람 하나하나 정말 말을 막힘없이 잘했다.
아쉬웠던 것은 하고 있는 업무가 있기 때문에 중간중간 전화를 받거나 회의를 하거나 계속해서 듣기가 어려웠다. 조금 집중해서 듣고 있으면 전화가 오는 경우가 많아서 안타까웠다.
SuperWeek의 공통 주제는 SuperApp에 대한 발표였다.
아직 정식출시는 조금 시간이 걸리겠지만 발표에 나온대로 개발만 된다면 편리한 생활을 누릴 수 있는 앱이 나오는 것이었다.

재미있었던 부분은 A-Book과 MyLife 였다.
A-Book 은 e-Book의 진화 형태로 단순한 텍스트와 2D 그림 형태의 전자 책자에서 3D 객체, 화면 이동, 손쉽게 만들어내는 템플릿 제공 등 책의 저자가 좀 더 독자에게 신선하고 재미있게, 생동감 있게 다가갈 수 있도록 많은 기능이 지원되는 것 같았다. 만드는 것 차제가 크게 어렵지 않아서 나도 저정도면 한장한장 써내려가다보면 책을 낼 수 있겠네? 라는 생각이 들 정도였다.
그리고 MyLife는 소개해주시는 분이 너무 웃겼다. 나긋나긋하고 귀에 쏙 박히는 목소리로 마치 A.I.가 말하는 듯한 방식으로 하나의 앱이 나를 분석한 데이터로 내 하루동안에 필요한 부분을 콕 집어 추천해주고 안내 해주는 앱의 형태를 보여주었다. 내가 SNS를 안하는 타입이라 주변 맛집이나 현재 유행하는 문화 같은 것을 잘 모르는데 굳이 찾아보고 알려고 노력하지 않아도 앱이 알아서 다 알려줬다. "Thanks. MyLife"

아쉬웠던 부분은 온라인 특성상 시연에 참가 할 수 없는 점이었다.
오프라인 관람의 경우 둘째날 점심시간 쯤에 시연해 볼 수 있는 부스가 마련되어 사용해볼 수 있었던거 같은데 온라인은 그저 세션에서 소개할 때 보여주는 시연을 바라 볼 수 밖에 없었다. 그래서 더더욱 오프라인 관람에 가지 못한게 아쉬웠다. 

물론 이 SuperWeek에서 발표된 것이 정작 출시되게 되면 세상에 얼마나 많은 관심을 받을지, 얼마나 완성된 모습으로 나올지는 모르겠지만 새로운 문화를 만들어 나가려고 하는 것과 남들이 시도하지 않은 갈을 손해를 보면서까지 도전하는 것에 박수를 보낸다.
나도 언젠 가는 저런 회사에서 일할 수 있을까하여 채용정보를 훑어보니 역시나 코딩테스트가 필요했고, 야근이 엄~~청 많은 것 같다. 
다음에도 이런 제작 발표회라던지 설명회가 있으면 일이 바쁘더라도 온라인으로 참여해서 시야를 넓혀가야겠다.
